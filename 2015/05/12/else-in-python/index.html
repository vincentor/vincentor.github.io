Python中的else

发表于 2015-05-11   |   分类于 开发   |  

很多语言的分支结构都会用if/else这种广为人知的语法结构，然而在python中，else不仅仅存在于if/else结构中，还有其它语法结构里面，也存在着鲜为人知的else分支。我在偶然的一次使用python的异常处理时，接触到了try里面的else分支，于是顺便就总结了一下Python里面的else语法结构。

  1. if/else
    if/else结构是最常用的一种语法结构，基本上从字面上的意思即可理解其含义。即if表示满足某些条件时做什么，else分支表示不满足的情况下执行的动作。示例代码如下：

    In [6]: a = 2; b = 3
In [7]: if a > b:
...:     print 'larger'
...: else:
...:     print 'less'
...:
less
In [8]:

  2. for/else
    python不同与其它语言的地方是它在循环结构中也可以用else，表示当循环正常结束时（即不是因为break语句而结束的循环），执行的分支。如下示例代码：

    In [8]: l = []
In [9]: for i in l:
   ...:     print i
   ...: else:
   ...:     print 'no elements'
   ...:
no elements

In [10]: l = [1,2]

In [11]: for i in l:
   ....:     print i
   ....: else:
   ....:     print 'end'
   ....:
1
2
end

In [12]: for i in range(3,10):
   ....:     print i
   ....: else:
   ....:     print 'end'
   ....:
3
4
5
6
7
8
9
end

  3. while/else
    while语句中的else子句，和for中的else语句相差不大，即当while语句中的判断条件为False时，执行else子句。示例代码如下：

    In [16]: i = 0
In [17]: while i < 3:
   ....:     print i
   ....:     i += 1
   ....: else:
   ....:     print 'end'
   ....:
0
1
2
end

  4. try-except/else
    try语句中的else子句与上述几种情况有较大差异，而且也更少用到。先看如下示例代码：

    for arg in sys.argv[1:]:
try:
    f = open(arg, 'r')
except IOError:
    print 'cannot open', arg
else:
    print arg, 'has', len(f.readlines()), 'lines'
    f.close()

    咋一看会让人觉得else的作用类似finally，但事实上并非如此。else表示当try-except之间未发生异常时才执行的分支。也许有人会问，如果想要这样直接把else中的语句放到try里面不就行了？但这样做会导致一个问题，如果else中的语句再引发一个异常（即f.readlines，也引发一个IOError），except中将无法区分所捕获的异常是否来自try块中的代码（即f=open(arg, ‘r’)）。因此在需要严格区分的情形，需要将不必要捕捉异常的代码放入else分之中而不是直接加到try块。我觉得这也是try-except/else不常用的原因（一般都直接放try块中了^_^）。而else和finally的区别，看如下示例代码即可一目了然：

    >>> def divide(x, y):
...     try:
...         result = x / y
...     except ZeroDivisionError:
...         print "division by zero!"
...     else:
...         print "result is", result
...     finally:
...         print "executing finally clause"
...
>>> divide(2, 1)
result is 2
executing finally clause
>>> divide(2, 0)
division by zero!
executing finally clause
>>> divide("2", "1")
executing finally clause
Traceback (most recent call last):
  File "<stdin>", line 1, in ?
  File "<stdin>", line 3, in divide
TypeError: unsupported operand type(s) for /: 'str' and 'str'

在使用django，jinjia等模板语言时，也会有在for循环之后有else分支，不知道是不是从python借鉴而来呢？
